E-RIHS Knowledge Base

The E-RIHS Knowledge Base is a repository of information about the E-RIHS infrastructure, services, and resources. It is designed to support the E-RIHS community in sharing and discovering information about the E-RIHS infrastructure, services, and resources.

The Knowledge Base is powered by Cordra, a platform for building and managing digital objects. Cordra is designed to support the creation, management, and sharing of digital objects, and to provide a flexible and extensible platform for building digital repositories.

Note: this user interface is primarily intended as an administrative user interface. Several user-friendly tools for end users are expected to be developed in the future, which will make underlying use of this Knowledgebase.

The E-RIHS Knowledge Base is a work in progress, and we welcome feedback and suggestions for improvement. If you have any questions, comments, or suggestions, please let us know.

Welcome new users!

  • Logging into this administration system, to directly add, modify and delete digital objects, is achieved using an ORCID iD.
  • However, users need to be pre-registered in the system before they can login; having an ORCID iD is not sufficient.
  • Registration is done by contacting the wim[dot]fremout[at]kikirpa[dot]be, specifying your ORCID iD and details of your primary affiliation/organisation.
    • Organisation details should include a full name, acronym (if applicable) and an official website url.
  • A user account will be set up for you, along with details of you as a person and your related organisation.

Registering a Service in the E-RIHS Knowledgebase

In addition to the key metadata directly describing an actual Service, the full definition of an E-RIHS Service is a combination of the individual definitions of a set of related entities (People, Tools, Techniques, etc), see model diagram, that all need to be defined first in the order described below:

Note: the links included here will only work for users logged int to the E-RIHS Knowledgebase.
  • Make sure you are registered in the system, see the previous tab, and login using your ORCID.
  • Optionally: Explore the technique list (text, diagram,or in the vocab system) to see if your required techniques are listed. There is an "other" option to add in or suggest new techniques.
  • Use the person form to define all of the Service team members. The organisation form can also be used if needed in relation to your team members. Please do search the system for existing person and organisation entries before adding in new ones.
  • Use the equipment and software forms to define the key tools used within your Service. This is required for FIXLAB, MOLAB and DIGILAB, but is optional for ARCHLAB services.
  • Use the method form to define, one or more methods/procedures which are followed by your Service. A method, defined here, is intended to define the default or normal approach and is intended to act as the starting point for the documentation of actual access projects.
  • Finally use the service form to define the actual service, referencing the related entities where required.
